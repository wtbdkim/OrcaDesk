# 폰 연동 — 3단계 테스트 (앱에 서버 내장 + 큐 공유)

3단계는 데스크탑 앱 안에 서버를 넣고, GUI와 폰이 **같은 큐**를 보게 만들었습니다.

## 큰 변화 (중요)
- 데스크탑 앱의 큐가 이제 "공유 큐(QueueStore)"를 씁니다. 화면은 1초마다
  이 큐를 폴링해서 갱신합니다. (계산 진행/폰이 추가한 것도 자동 반영)
- Settings 탭에 **"Phone sync (beta)"** 카드가 생겼습니다.

## A. 먼저 데스크탑 앱이 그대로 잘 되는지 (회귀 확인)
빌드해서 ORCAdesk.exe 실행 후:
1. Build 탭에서 계산 추가 → Queue 탭에 뜨는지
2. Run queue → Log 탭에 ORCA 출력 흐르고, Queue 상태가 done까지 가는지
3. 결과/편집/삭제가 예전처럼 되는지
→ 이게 안 되면 큐 공유 전환에 문제가 있는 것이니 바로 알려주세요.
   (직전 백업: ORCAdesk_backup_before_stage3.zip 로 롤백 가능)

## B. 서버 켜기 (폰 연동)
1. (한 번만) 서버 의존성 설치:  pip install -r requirements-server.txt
2. 앱 실행 → Settings 탭 → "Phone sync (beta)" → **Start server**
3. "Running — open this address on your phone:" 와 주소(예: http://192.168.0.12:8000)가 뜨면 성공
   - "Not available" 이 뜨면 1번(의존성 설치)이 안 된 것.

## C. 같은 큐 공유 확인 (핵심!)
PC와 폰이 **같은 Wi-Fi**에 있어야 합니다.
1. 폰 브라우저에서 그 주소 열기 (예: http://192.168.0.12:8000/docs 또는 /api/queue)
2. 폰(또는 다른 브라우저)에서 POST /api/queue 로 계산 추가
3. → **데스크탑 앱 Queue 탭에 그 계산이 1초 안에 나타나는지** 확인
4. 반대로 데스크탑에서 추가 → 폰에서 /api/queue 새로고침 시 보이는지
5. 데스크탑에서 Run → 폰에서 /api/log, /api/queue 로 진행 상황 보이는지

이게 되면 "PC와 폰이 한 큐를 공유" = 3단계 목표 달성입니다.

## 주의
- 앱을 닫으면 서버도 같이 꺼집니다.
- 아직 인증/QR/외부접속(터널)은 없습니다 — 같은 Wi-Fi 안에서만, 주소 직접 입력.
  (5단계에서 QR + Cloudflare 터널, 6단계에서 토큰 인증)
- 기숙사 공용 Wi-Fi라면 같은 망의 타인도 주소를 알면 접속할 수 있으니,
  지금 단계에선 테스트만 하고 평소엔 서버를 꺼두는 걸 권장합니다.

문제 생기면 어느 단계(A/B/C)에서 막혔는지 + 화면/에러를 알려주세요.
